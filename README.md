## Working Style

- Think in user-facing outcomes, not implementation steps.
- Keep features atomic so AI can deliver in one pass and humans can review in minutes.
- Sync code with feature docs (`features/*.md`, `features/docs/*.md`, `features/apis/*.md`) after changes.

For the complete mindset shift and examples, see [ai-feature-driven-development.md](ai-feature-driven-development.md).
